# Veridian (AI Notary) — On-Chain Truth Machine on GenLayer

**Veridian (AI Notary)** is a decentralized application (dApp) that leverages GenLayer's Intelligent Contracts to notarize claims about the real world using web content. Users submit any public URL and a natural language claim about its content. The contract fetches the web page on-chain, utilizes decentralized LLMs to reason over the text, and writes a verifiable attestation record directly to the ledger.

Unlike traditional smart contracts, Veridian requires **no oracles** or off-chain scripts. GenLayer's validators fetch the data and execute LLM prompts natively, reaching agreement on subjective outcomes using the **Equivalence Principle**.

##  Architecture & How It Works

```
┌──────────────────────┐      JSON-RPC / SDK       ┌─────────────────────────┐
│   index.html         │  ◄──────────────────────►  │   GenLayer Testnet      │
│   (Browser dApp)     │                            │                         │
│   • Virtual Account  │                            │   notary.py             │
│   • GenLayer JS SDK  │                            │   • On-chain web fetch  │
│   • Tailwind & CSS   │                            │   • LLM prompt consensus│
│                      │                            │   • DynArray storage    │
└──────────────────────┘                            └─────────────────────────┘
```

1. **Transaction Submission**: The client submits a transaction calling the `notarize(url, claim)` method.
2. **On-Chain Web Fetch**: GenLayer validators call `gl.nondet.web.render(url, mode="text")` to download and clean the target web page.
3. **Decentralized LLM Reasoning**: The contract runs `gl.nondet.exec_prompt(prompt, response_format="json")` to analyze the page content against the claim.
4. **Equivalence Principle**: Since LLM outputs can vary slightly in formatting and wording, validators achieve consensus via `gl.eq_principle.prompt_comparative` to ensure their verdicts, confidence levels, and summaries align before committing the block.
5. **Ledger Record**: Once consensus is reached, the transaction is finalized, and the JSON-serialized attestation is appended to the contract's `DynArray[str]` state.

## Deployment & Local Setup

### 1. Deploy the Contract on GenLayer Studio
1. Open [GenLayer Studio](https://studio.genlayer.com/)
2. Create a new contract file and paste the entire contents of **`notary.py`**.
3. Under the deployment tab, click **Deploy** to compile and publish the contract to the GenLayer testnet.
4. Copy your **deployed contract address** (e.g., `0xAbCdEf...1234`).

### 2. Configure the Frontend
1. Open **`index.html`** in a text editor.
2. Locate the contract configuration near the top of the `<script>` tag:
   ```javascript
   const CONTRACT_ADDRESS = "PASTE_YOUR_CONTRACT_ADDRESS_HERE";
   ```
3. Replace the placeholder string with your deployed contract address and save.

### 3. Run Locally
Because the frontend imports `genlayer-js` using ES Modules via `esm.sh`, it must be served through an HTTP server (not opened as a local file double-clicked via `file://`).

Start a simple HTTP server in the repository directory:
```bash
# Python 3
python -m http.server 8000

# Node.js / npx
npx serve .
```

Open `http://localhost:8000` (or `http://localhost:3000`) in your browser. The application will automatically generate a virtual account for you to send transactions and query attestations.
